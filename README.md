# Wheel_Maker
Wheel_Maker 是一个专为机器人底盘设计的性能评估与轮组选型工具。本项目通过运动学与动力学联合解算，帮助硬件工程师和算法工程师在设计初期快速评估底盘的理论性能极限、电机扭矩需求以及抓地力安全边界。

⚠️ 当前版本状态：目前 Wheel_Maker 仅支持 四轮舵轮（Swerve Drive）底盘 的计算与评估。
🚀 后续更新规划：未来版本将陆续加入对 轮腿结合构型、麦克纳姆轮（Mecanum）、全向轮（Omni）等其他底盘构型的支持。

📂 文件结构与架构

项目主要由以下四个 MATLAB 脚本构成：

main.m - 项目主入口，负责调度配置参数、执行工况模拟并输出最终的性能评估报告。

Config_Params.m - 全局配置字典，集中管理机器人的所有物理参数、环境常数和目标机动指标。

Dynamics_Core.m - 动力学核心联合引擎，负责载荷转移计算、阻力分析、扭矩分配及打滑校验。

Kinematics_Solver.m - 运动学解算内核，负责底盘速度到单轮状态的逆解计算，包含最优转向逻辑。

⚙️ 工作原理详解

Wheel_Maker 的核心工作流分为 参数配置、运动学解算 和 动力学评估 三个维度：

1. 全局参数化配置 (Config_Params.m)

系统将所有变量抽象为五个维度的常量：

几何与质量：轮距、轮径、重心高度、整车质量等。

传动与电机：减速比、驱动/转向电机额定转速。

极限机动指标：预期最大平移加速度、最大瞬态角加速度。

环境经验常数：地面摩擦系数、地胶阻力放大系数、机械损耗。

材料属性：轮毂包胶的邵氏硬度（Shore A）以及安全冗余系数。

2. 运动学逆解与优化 (Kinematics_Solver.m)

该模块将底盘的目标运动矢量 $(v_x, v_y, \omega_z)$ 转化为四个舵轮的控制指令。

基础逆解：根据轮子在底盘的相对几何位置，计算每个轮子所需的速度大小和方向。

⭐ 最优转向逻辑 (优角转劣角)：当目标偏转角与当前轮向的角度差超过 90°（$\pi/2$）时，算法会自动将目标角度反转 180°，并同时反转驱动电机的速度方向。这极大缩短了舵向的响应时间，是舵轮底盘控制的经典且必要的优化。

3. 动力学核心引擎 (Dynamics_Core.m)

这是评估底盘受力极限的最关键模块：

材料建模：引入了经验公式，将用户输入的“邵氏硬度”转化为“杨氏模量 (E)”，为后续的滚阻计算提供物理依据。

动态载荷转移 (Weight Transfer)：当底盘发生高加减速时（如极限爆发工况），根据重心高度（h_cog）计算前后左右四个轮子上的动态垂直载荷变化（$F_z$）。

力矩合成：

驱动扭矩：不仅考虑了加速所需的牵引力，还综合了由于轮子形变产生的纯滚阻、地胶放大系数以及机械摩擦。

转向扭矩：考虑了地面摩擦阻力矩（与动态法向力正相关）、机械阻力以及克服转向模块转动惯量所需的扭矩。

抓地力安全校验：基于库仑摩擦力模型（摩擦圆/摩擦椭圆），校验当前分配给单轮的合力是否超过了该轮当前法向力提供的最大静摩擦力。如果超过，则触发 Slip_Warning 打滑警告。

4. 工况模拟与报告输出 (main.m)

主程序会提取配置参数，进行两次核心计算：

基于物理边界的运动学正解：计算出在当前电机转速和减速比下，底盘的理论最高直线速度和最高自转角速度。

基于恶劣工况的动力学逆解：

巡航工况：加速度 $a=0$ 时的持续扭矩需求（用于评估电机额定发热）。

极限爆发工况：模拟底盘沿对角线 45° 满载加速，结合满载转向角加速。这是对四个轮子法向力分布最极端的考验。

🚀 使用指南

步骤 1：调整配置参数

使用前，请首先打开 Config_Params.m 文件。根据你当前设计的机器人 CAD 模型和选型的电机规格，修改对应的属性值。

提示：特别是 h_cog (重心高度) 和 m_total (总重) 对极限工况下的打滑评估影响巨大。

步骤 2：运行主程序

在 MATLAB 环境中，直接运行 main.m 文件。

步骤 3：阅读性能评估报告

程序运行后，MATLAB 命令行窗口将输出格式化的评估报告：

【运动表现表现】：告诉你这套轮组能跑多快、转多快。用来对照赛规或战术需求。

【电机轴端扭矩需求】：直接给出驱动和转向电机在持续和峰值状态下的扭矩需求（已包含你设定的安全冗余和减速比）。拿着这个数据去对比电机供应商的特性曲线（T-N 曲线）即可完成电机选型。

【抓地力安全评估】：如果提示 [危险] 检测到车轮打滑！，说明你期望的 target_max_a (加速度) 过高，或者重心太高导致某侧轮子离地/失去抓地力。你需要降低预期加速度，或者在机械上降低重心、增加轴距。
